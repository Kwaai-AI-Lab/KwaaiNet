## Answers

### q01 — What is a "Changeset" in OpenStreetMap, and what purpose does it serve when editing map data?

A Changeset in OpenStreetMap (OSM) is a group of edits made by a single user over a short period of time. It represents the collective changes made to the database as a whole, rather than individual elements like nodes or ways.

According to source [2], "A changeset is a group of edits to the database by a single user over a short period of time." The same source further explains that this changeset can include multiple edits such as adding new objects (like ways), deleting areas (e.g., forest removed for houses), or changing tags on existing elements.

Changesets serve several purposes when editing map data in OSM:

1.  **Grouping related edits**: A Changeset groups multiple related edits made by a user, allowing the system to track and attribute these changes to the specific user.
2.  **Version control**: OSM's version control system retains old versions of data and associates them with users through Changesets, enabling easy tracking and reverting of changes.
3.  **Editing history**: Each Changeset provides a snapshot of edits made at a particular point in time, facilitating the understanding of editing activities and identifying potential conflicts or issues.

By grouping related edits into Changesets, OSM ensures that all modifications to map data are properly tracked, assigned to users, and stored as part of the database's version control system.

### q02 — According to the Elements page, what are the three fundamental data primitives used to represent geographic data in OpenStreetMap?

The three fundamental data primitives used to represent geographic data in OpenStreetMap are:

1. Nodes (deﬁning points in space)
2. Ways (deﬁning linear features and area boundaries)
3. Relations (deﬁning how other elements work together) [7]

These are mentioned as the "three core data elements of the OpenStreetMap data model."

### q03 — What is a "Node" in OpenStreetMap's data model, and what does it represent?

A node represents a specific point on the earth's surface defined by its latitude and longitude, referred to the World Geodetic System 1984 [1]. Each node comprises at least an id number and a pair of coordinates. A node can represent a point of interest or an object of minuscule extent [6]. Because nodes are the only type of object that has a coordinate, most of the nodes serve only as a coordinate for an intermediate point within a way and carry no tags [6].

A node is one of the core elements in the OpenStreetMap data model. It consists of a single point in space defined by its latitude, longitude and node id [19]. A third, optional dimension (altitude) can also be included: key:ele (abbreviated for "elevation") [19].

### q04 — What is a "Way" in OpenStreetMap, and how does it relate to nodes?

According to excerpts [3] and [16], a Way in OpenStreetMap is an ordered list of between 1 (!) and 2,000 nodes that define a polyline. Ways are used to represent linear features such as rivers and roads.

A node can be included as a member of a way, and the way refers to the nodes by their IDs. The way gets its geometry by using the coordinates of the referenced nodes.

### q06 — What is a "Tag" (key-value pair) in OpenStreetMap, and how is it used to describe the meaning of map elements?

A tag in OpenStreetMap is a key-value pair that describes specific properties of map features. It consists of two free-format text fields: a key and a value, separated by an equals sign (=). Both parts are Unicode strings of up to 255 characters [6]. For example, "highway=residential" defines the way as a road whose main function is to give access to people's homes.

Tags can be used to describe various properties of map elements, such as:

* The type of feature (e.g., highway, building, or water)
* Its name or address
* Additional information about its purpose or characteristics

Each map element can have multiple tags, and each key must be unique [6]. Some tags may also include additional syntax for alternative keys or values, such as using a semi-colon to separate multiple values [13].

Conventions are agreed upon the meaning and use of tags, which are captured on the OpenStreetMap wiki. In some cases, there may be competing visions about what tags actually mean, or mappers may use them in different ways [2].
